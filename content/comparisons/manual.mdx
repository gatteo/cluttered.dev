## Overview

You're a developer. You know how to delete folders. Why would you need a tool for that?

It's a fair question. Let's break down the real cost of manual cleanup versus using Cluttered.

## The Manual Approach

Here's what manual cleanup typically looks like:

```bash
# Find node_modules
find ~/projects -name "node_modules" -type d

# Check each one manually
cd ~/projects/old-project
ls -la node_modules  # How big is it?
git status           # Any uncommitted changes?
stat package.json    # When was this last modified?

# Decide and delete
rm -rf node_modules

# Repeat for every project...
```

## Ecosystems Multiply the Problem

Manual cleanup for one ecosystem is tedious. Now multiply by 12:

- node_modules (npm, yarn, pnpm)
- target/ (Rust)
- DerivedData (Xcode)
- venv (Python)
- build/ (Gradle, Maven)
- vendor/ (Go, Ruby, PHP)
- ...and more

Each ecosystem has different:
- Directory names
- Safe deletion patterns
- Detection of active projects
- Restoration commands

## What Manual Cleanup Misses

| Factor | Manual | Cluttered |
|--------|--------|-----------|
| Hidden directories | Often missed | Always found |
| Nested projects | Requires recursive search | Automatic detection |
| Project activity | Manual git status | Instant indicators |
| Space tracking | Manual calculation | Running totals |
| History | No record | Full log |

## The Real Risk

Manual cleanup has higher error potential:

- **Wrong directory**: `rm -rf` in the wrong folder is catastrophic
- **Active project**: Deleting while building causes issues
- **No undo**: Gone means gone
- **Muscle memory**: Dangerous when tired

Cluttered prevents these by:
- Showing clear project names and paths
- Detecting active builds and git status
- Moving to Trash before permanent deletion
- Providing full history and undo

## When Manual Makes Sense

Manual cleanup is fine if:

- You have only 1-2 projects
- You work in a single ecosystem
- You enjoy terminal archaeology
- You never make mistakes while tired

## When Cluttered Makes Sense

Cluttered pays for itself if:

- You have more than 5 projects
- You work across multiple ecosystems
- You value your time
- You want safety rails
